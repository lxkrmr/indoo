# Contributing

## Commits

Use Conventional Commits.

Format:

```text
type(scope): short description
```

Examples:

```text
feat(cli): add list command with safe pagination
fix(output): keep text summaries stable
docs(readme): simplify quickstart
refactor(client): extract payload parsing
test(cli): cover list defaults
```

Rules:
- keep commits small and meaningful
- write commit messages in English
- prefer one focused change per commit
- use a scope that matches the main area you changed

Common types:
- `feat`
- `fix`
- `docs`
- `refactor`
- `test`
- `chore`
